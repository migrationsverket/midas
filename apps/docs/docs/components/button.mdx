Komponent som används för att låta användaren utföra en handling t.ex. spara ifylld information eller öppna ett formulär.

```tsx
import { Button, ButtonGroup } from '@midas-ds/components'
```

```tsx
<ButtonGroup aria-label='Välj knapp'>
  <Button>Primary</Button>
  <Button variant='secondary'>Secondary</Button>
  <Button variant='tertiary'>Tertiary</Button>
  <Button isDisabled>Disabled</Button>
</ButtonGroup>
```

<div className='card'>
  <ButtonGroup aria-label='Välj knapp'>
    <Button>Primary</Button>
    <Button variant='secondary'>Secondary</Button>
    <Button variant='tertiary'>Tertiary</Button>
    <Button isDisabled>Disabled</Button>
  </ButtonGroup>
</div>

## Varianter

### Primär

En primär knapp används för den primära eller positiva handlingen i ett flöde (OK, Skicka, Nästa). Det bör endast finnas en primärknapp per sida, sektion eller tydligt avgränsad yta.

```tsx
<Button>Slutför</Button>
```

<div
  className='card'
  style={{ display: 'block' }}
>
  <Button>Slutför</Button>
</div>

### Sekundär

En sekundär knapp används för underordnade handlingar i ett flöde (Avbryt, Tillbaka). Används alltid tillsammans med en primär knapp.

```tsx
<ButtonGroup aria-label='Välj knapp'>
  <Button>Slutför</Button>
  <Button variant='secondary'>Tillbaka</Button>
</ButtonGroup>
```

<div className='card'>
  <ButtonGroup aria-label='Välj knapp'>
    <Button>Slutför</Button>
    <Button variant='secondary'>Tillbaka</Button>
  </ButtonGroup>
</div>

### Tertiär

Den tertiära knappen kan användas tillsammans med primär- och sekundär knapp där det finns behov av ett ytterligare val

```tsx
<ButtonGroup aria-label='Hantera anmälan'>
  <Button>Slutför</Button>
  <Button variant='secondary'>Tillbaka</Button>
  <Button variant='tertiary'>Ta bort anmälan</Button>
</ButtonGroup>
```

<div className='card'>
  <ButtonGroup aria-label='Hantera anmälan'>
    <Button>Slutför</Button>
    <Button variant='secondary'>Tillbaka</Button>
    <Button variant='tertiary'>Ta bort anmälan</Button>
  </ButtonGroup>
</div>

Om den tertiära knappen används fristående från andra knappar ska den ha en ikon. Ikonen läggs till via `icon`.
Du kan välja vilken sida av texten ikonen ska vara på med `iconPlacement`.

```tsx
<Button
  variant='tertiary'
  // highlight-start
  icon={Plus}
  iconPlacement='left'
  // highlight-end
>
  Lägg till rad
</Button>
```

<div
  className='card'
  style={{ display: 'block' }}
>
  <Button
    icon={Plus}
    variant='tertiary'
    iconPlacement='left'
  >
    {'Lägg till rad'}
  </Button>
</div>

## Grupp av knappar

För att få korrekt layout på flertalet knappar kan komponenten `ButtonGroup` användas.

```tsx
import { ButtonGroup } from '@midas-ds/components'
```

```tsx
<ButtonGroup aria-label='Välj en frukt'>
  <Button>Pineapple</Button>
  <Button variant='secondary'>Apple</Button>
  <Button isDisabled>Orange</Button>
</ButtonGroup>
```

<div className='card'>
  <ButtonGroup aria-label='Välj en frukt'>
    <Button>Pineapple</Button>
    <Button variant='secondary'>Apple</Button>
    <Button isDisabled>Orange</Button>
  </ButtonGroup>
</div>

### Label

För god tillgänglighet bör du sätta en aria-label på `ButtonGroup` för att beskriva vad knapparna gör.

```tsx
// highlight-start
<ButtonGroup aria-label='Välj en frukt'>
  // highlight-end
  <Button>Pear</Button>
  <Button variant='secondary'>Apple</Button>
  <Button isDisabled>Grape</Button>
</ButtonGroup>
```

<div className='card'>
  <ButtonGroup aria-label='Välj en frukt'>
    <Button>Pear</Button>
    <Button variant='secondary'>Apple</Button>
    <Button isDisabled>Grape</Button>
  </ButtonGroup>
</div>

### Kontrollera flertalet knappar

För att kontrollera flertalet knappar kan `ButtonContext` användas runtom flertalet knappar för att kontrollera attribut på samtliga knappar på ett ställe.

```tsx
import { ButtonContext } from 'react-aria-components'
```

```tsx
// highlight-start
<ButtonContext.Provider value={{ isDisabled: true }}>
  // highlight-end
  <ButtonGroup>
    <Button>Grape</Button>
    <Button variant='secondary'>Pear</Button>
    <Button variant='tertiary'>Banana</Button>
  </ButtonGroup>
  // highlight-start
</ButtonContext.Provider>
// highlight-end
```

<div className='card'>
  <ButtonContext.Provider value={{ isDisabled: true }}>
    <ButtonGroup>
      <Button>Grape</Button>
      <Button variant='secondary'>Pear</Button>
      <Button variant='tertiary'>Banana</Button>
    </ButtonGroup>
  </ButtonContext.Provider>
</div>

## Riktlinjer

### Val av komponent

Se mönstret [Knappar och länkar](/design-patterns/buttons-and-links)

### Generella riktlinjer

- Undvik inaktiva knappar. Alla knappar i externa system ska gå att interagera med. Om användaren inte uppfyllt kraven, t.ex för att gå till nästa sida i en e-tjänst, ska ett felmeddelande visas som påtalar vad som krävs för att kunna gå vidare.
- Undvik att ha för många knappar på samma sida.

### Placering och ordning

Knappar placeras i regel i nedre vänstra hörnet. Vi sätter alltid den primära knappen till vänster.

```tsx
<ButtonGroup aria-label='Skicka formulär'>
  <Button>Skicka</Button>
  <Button variant='secondary'>Avbryt</Button>
</ButtonGroup>
```

<div className='card'>
  <ButtonGroup aria-label='Skicka formulär'>
    <Button>Skicka</Button>
    <Button variant='secondary'>Avbryt</Button>
  </ButtonGroup>
</div>
