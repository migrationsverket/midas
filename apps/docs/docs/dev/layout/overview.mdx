:::caution Beta

`@midas-ds/layout` är i beta. Paketet är installationsbart och redo att användas, men visuell design och vissa API:er kan förändras innan stabil release.

:::

`@midas-ds/layout` är ett modulärt layoutpaket som ersätter `Layout` från `@midas-ds/components`. Om du använder det gamla paketet idag så kan du följa [migrationsguiden](/dev/layout/migration) för att gå över till det nya paketet.

## Installation

```bash
npm install @midas-ds/layout
```

Importera `default.css`. Denna fil innehåller global css för att layout-komponenterna ska få rätt utseende. Importera filen i roten av din applikation:

```tsx title="main.tsx"
import '@midas-ds/layout/default.css'
```

## Val av sidlayout

Tabellen nedan beskriver vilken sidlayout som ska användas för olika typer av applikationer.

| Typ av applikation                                                 | Sidlayout                                                                      |
| ------------------------------------------------------------------ | ------------------------------------------------------------------------------ |
| En enkel one-page applikation utan navigation (t.ex. ett formulär) | [Endast Header](/design-patterns/page-layout/only-header)                      |
| Intern desktopapplikation                                          | [Header, Sidebar och Panel](/design-patterns/page-layout/header-sidenar-panel) |
| Extern desktopapplikation                                          | [Header, Sidebar och Panel](/design-patterns/page-layout/header-sidenar-panel) |
| Extern applikation som ska upplevas som en mobilapp                | [Header, Navbar och Panel](/design-patterns/page-layout/header-sidenar-panel)  |

## Exempelapp

Vi håller på att ta fram en fristående exempelapp med Vite som visar paketet i ett realistiskt sammanhang. Under tiden finns [`apps/next`](https://github.com/migrationsverket/midas/tree/main/apps/next) i monorepon, en Next.js-app som använder paketet fullt ut och är öppen att titta på för inspiration.

:::info Next.js / SSR

Vissa komponenter i paketet (bl.a. `Header`) kräver `'use client'` i Next.js App Router. Använd dem i klientkomponenter eller wrappa dem i en egen `'use client'`-komponent. Se [`apps/next`](https://github.com/migrationsverket/midas/tree/main/apps/next) för ett fungerande exempel.

:::
