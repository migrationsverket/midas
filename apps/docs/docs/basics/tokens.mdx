Här kan du läsa om hur vi använder färger i Midas och hur de är tokeniserade. Tokens är en del av vår design och används för att skapa enhetlighet i våra produkter. Vi har definierat ett antal paletter som används i designsystemets komponenter.

<div style={{ display: 'flex', flexFlow: 'row wrap', marginBottom: '2rem' }}>
  {Object.entries(baseColors)
    .filter(([key]) => key.startsWith('gray') || key.startsWith('blue') || key.startsWith('signal'))
    .reduce((acc, [key, value], index, array) => {
      const prefix = key.split(/(?=[A-Z0-9])/)[0];
      const lastPrefix = index > 0 ? array[index - 1][0].split(/(?=[A-Z0-9])/)[0] : null;

      if (prefix !== lastPrefix) {
        acc.push(<div style={{ flexBasis: '100%' }} key={`break-${index}`} />);
      }

      acc.push(
        <div
          style={{ backgroundColor: value, width: 60, height: 60 }}
          key={key}
        />
      );

      return acc;
    }, [])}

</div>

## Referenser och semantiska tokens

Våra färgtokens är uppdelade i två grupper: referenstokens och semantiska tokens.

### Referenstokens

Referenstokens är direkt kopplade till färgvärden och används för att bygga upp våra paletter. Varje palett har 20 steg, från 10 (ljusast) ill 200 (mörkast). Referenstokens används endast som byggstenar och ska inte användas direkt i koden.

### Semantiska tokens

Semantiska tokens är kopplade till referenstokens. De är döpta för att beskriva vart de bör användas.
används för att sätta en färg på element i både ljust och mörk läge. Nedan ser vi ett enkelt exempel med bakgrund och text med ett token men olika färger i ljust respektive mörkt läge.

## Lager

Färgerna är strukturerade i lager. Detta gör det lättare att förstå hur de ska användas. Varje lager representerar en nivå av information eller innehåll. Det första lagret är det mest grundläggande och används för bakgrunder, medan de högre lagren används för text och andra element.
Det finns två lager: layer01 och layer02. Dessa lager används för att skapa djup och hierarki i designen. De används också för att separera olika delar av gränssnittet.

Båda lagren har även accentfärger som används för att markera viktiga element eller interaktioner. Dessa accentfärger är också namngivna med prefixet "layer" följt av "Accent" och en siffra som anger vilket lager de tillhör. Till exempel `layerAccent02` används för att markera viktiga element på det andra lagret.

## Namngivning

Tokens är namngivna med en prefix som anger deras syfte. Prefixet kan vara "layer", "text", "border" eller "shadow". Detta gör det lättare att förstå hur och var de ska användas.

export const Token = ({token, desc}) => {
    const tokenValue = semantic[token]

    if (tokenValue === undefined) {
        console.warn(`Token ${token} not found in semantic tokens`)
        return null
    }

    return (
        <FlexItem col={12} style={{ borderBottom: '1px solid var(--ifm-color-emphasis-200)', paddingBottom: '1rem' }}>
            <Flex fluid>
                <FlexItem col={3}>
                    <small style={{wordBreak: 'break-all'}}>{token}</small>
                </FlexItem>
                <FlexItem col={3}>
                    <small>{desc}</small>
                </FlexItem>
                <FlexItem col={6} style={{alignItems: 'flex-end'}}>
                    <div style={{ display: 'flex', flexDirection: 'row', gap: '0.5rem' }}>
                        <div style={{display: 'flex', flexDirection: 'column'}}>
                            <small>Ljust: #{tokenValue.match(/light-dark\(#(.*?),/)[1]}</small>
                            <small>Mörkt: #{tokenValue.match(/light-dark\(.*?, #(.*?)\)/)[1]}</small>
                        </div>
                        <div style={{ colorScheme: 'light', backgroundColor: tokenValue, width: 60, height: 60, border: '1px solid var(--ifm-color-emphasis-100)' }} />
                        <div style={{ colorScheme: 'dark', backgroundColor: tokenValue, width: 60, height: 60, border: '1px solid var(--ifm-color-emphasis-100)' }} />
                    </div>
                </FlexItem>
            </Flex>
        </FlexItem>
    )

}
