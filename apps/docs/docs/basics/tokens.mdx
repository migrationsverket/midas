Här kan du läsa om hur vi använder färger i Midas. Vi använder i huvudsak en gråskala och en blå skala som grund för
våra gränssnitt. Gråskalan används främst för bakgrunder, ramar och typografi, medan den blå skalan fungerar som vår
primära accentfärg för interaktiva element. Genom att bygga gränssnitt med dessa två paletter skapar vi ett tydligt,
tillgängligt och konsekvent uttryck i hela designsystemet. För att säkerställa konsekvent användning av färger använder vi designtokens, variabler som representerar färgernas syfte
snarare än deras exakta värde.

## Kom igång

Färgtokens är tillgängliga som typescriptvariabler via:

```tsx
import { variables } from '@midas-ds/theme'
```

Det går också att använda CSS `var(--variable-name)` genom att importera `theme.css`. Mer information om
hur du använder Midas tokens i CSS hittar du under [Installera Midas](/get-started/install/).

```tsx
import '@midas-ds/theme/variables.css'
```

```css
.my-class {
  color: var(--midas-text-on-color); /* textOnColor */
}
```

## Användning

:::info
I de fall där systemet inte kan använda tokens så finns en [fullständig tokenreferens med färgkoder här](/get-started/contribute/tokens/dev-tokens).
:::

Eftersom komponenterna redan är byggda med tokens behöver vi generellt bara använda de tokens som är bundna till UI-element som inte är komponenter. Dessa är beskrivna i [tokenlistan](#tokenlista).

Våra semantiska tokens är döpta för att beskriva var de bör användas. Varje semantisk token är kopplad till två referenstokens, en för ljust och en för mörkt läge, vilket gör att vi kan byta färgtema utan att behöva ändra något i kod. Det är därför viktigt att använda tokens efter syfte snarare än efter färg.

### Lager

Vissa färgtokens är kopplade till ett specifikt lager, vilket indikeras av att de har 01 eller 02 i namnet. Numreringen används för att avgöra
vilka tokens som ska användas beroende på vilket lager en komponent ligger på. Tokens som har 01 i namnet (t.ex. `field01Base`)
används när en komponent ligger på `backgroundBase` och tokens som har 02 i namnet (t.ex. `field02Base`) används när komponenten ligger på `layer01Base`.

I ljust läge är varannat lager vitt och varannat grått. I mörkt läge blir varje lager ljusare, se exempel nedan.
